## [23. 合并 K 个升序链表](https://leetcode.cn/problems/merge-k-sorted-lists/)

**小顶堆（优先队列）**

把每个链表的头节点放进堆，每次弹出最小节点，把它的 `next` 再放进堆。

Python 的 `heapq` 比较对象时需要可比较；我们用 `(val, idx, node)` 这种 tuple

注意：`idx` 用来避免 val 相等时比较 node 报错。

```python
from typing import Optional
import heapq
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next
class Solution:
    def mergeKLists(self, lists: List[Optional[ListNode]]) -> Optional[ListNode]:
        heap = [] # 堆 就是一个列表
        for i, node in enumerate(lists):# 把每个链表的头节点放进堆，遍历用enumerate
            if node:
                heapq.heappush(heap, (node.val, i, node))# 注意语法

        dummy = ListNode(0)
        tail = dummy

        while heap:
            _, i, node = heapq.heappop(heap)# 弹出最小节点，这个i是轮转使用的（一共就这几个）
            tail.next = node
            tail = tail.next

            if node.next:# 把它的 `next` 再放进堆
                heapq.heappush(heap, (node.next.val, i, node.next))
        return dummy.next
```

## [148. 排序链表](https://leetcode.cn/problems/sort-list/)

**自顶向下归并（递归版，最容易写对）**

思路：

1. 用快慢指针找中点，把链表断成两半
2. 递归排序左右两半
3. 合并两个有序链表

```python
from typing import Optional
class Solution:
    def sortList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if not head or not head.next:
            return head
        # 1) 找中点并断开
        slow, fast = head, head.next
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
        mid = slow.next
        slow.next = None  # 断开
        left = self.sortList(head)
        right = self.sortList(mid)
        return self._merge(left, right)
	
    # 普通的合并函数
    def _merge(self, a: Optional[ListNode], b: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0)
        tail = dummy
        while a and b:
            if a.val <= b.val:
                tail.next = a
                a = a.next
            else:
                tail.next = b
                b = b.next
            tail = tail.next # 这句不要忘
        tail.next = a if a else b
        return dummy.next
```

## [153. 寻找旋转排序数组中的最小值](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/)

旋转数组形如：`[4,5,6,7,0,1,2]`，最小值在“断点”处。

用二分时最常用的判断是比较 `nums[mid]` 和 `nums[r]`：

- 如果 `nums[mid] > nums[r]`：说明最小值在 **右半边**（mid 右侧），`l = mid + 1`
- 否则：说明最小值在 **左半边或 mid 本身**，`r = mid`

最终 `l == r`，就是最小值位置。

------

代码（最推荐模板：左闭右闭 + r=mid）

```python
from typing import List

class Solution:
    def findMin(self, nums: List[int]) -> int:
        l, r = 0, len(nums) - 1
        while l < r:  	# l = r 时就返回了，跳出循环
            mid = (l + r) // 2
            if nums[mid] > nums[r]:
                l = mid + 1
            else:
                r = mid
        return nums[l]
```

------

为什么 `r = mid` 不用 `mid - 1`？

因为当 `nums[mid] <= nums[r]` 时，`mid` **有可能就是最小值**，不能排除它，所以右边界收缩到 `mid` 而不是 `mid-1`。

## [394. 字符串解码](https://leetcode.cn/problems/decode-string/)

```python
输入：s = "3[a2[c]]"
输出："accaccacc"
```

遍历字符串 `s`，维护：

- `num`：当前正在解析的重复次数（可能是多位数）
- `cur`：当前层正在构建的字符串
- `stack`：遇到 `[` 时，把 “**上一层字符串 + 次数**” 压栈，进入新层

规则：

1. 遇到数字：累积 `num = num * 10 + int(c)`
2. 遇到 `[`：把 `(cur, num)` 压栈，重置 `cur=""`、`num=0`
3. 遇到 `]`：弹出 `(prev, k)`，令 `cur = prev + cur * k`
4. 遇到字母：`cur += c`

```python
class Solution:
    def decodeString(self, s: str) -> str:
        stack = []  # 存 (prev_str, repeat_k)
        num = 0
        cur = ""

        for c in s:
            if c.isdigit():
                num = num * 10 + int(c)

            elif c == '[':
                stack.append((cur, num))
                cur = ""
                num = 0

            elif c == ']':
                prev, k = stack.pop()
                cur = prev + cur * k

            else:  # 字母用else（更简洁）
                cur += c

        return cur
```

```python
s = "3[a2[c]]"
```

- 读到 `3`：num=3
- `[`：压栈 ("",3)，cur=""
- `a`：cur="a"
- `2`：num=2
- `[`：压栈 ("a",2)，cur=""
- `c`：cur="c"
- `]`：弹 ("a",2) → cur="a"+"c"*2="acc"
- `]`：弹 ("",3) → cur=""+"acc"*3="accaccacc"

## [84. 柱状图中最大的矩形](https://leetcode.cn/problems/largest-rectangle-in-histogram/)

**单调栈**

对每根柱子高度 `h[i]`，它能扩展成的最大矩形宽度，取决于：

- 左边第一个 **严格小于** `h[i]` 的位置 `L`
- 右边第一个 **严格小于** `h[i]` 的位置 `R`

那么以 `h[i]` 为高的最大面积：
area = h[i] * (R - L - 1)

单调递增栈可以在一次扫描里找到这些边界。

------

代码（推荐：加哨兵，最不容易错）

技巧：在数组两端加 0（哨兵），保证所有柱子都会被弹栈计算面积。

```python
from typing import List

class Solution:
    def largestRectangleArea(self, heights: List[int]) -> int:
        h = [0] + heights + [0]   # 两端哨兵
        stack = [0]               # 存索引，保持 h[stack] 单调递增
        ans = 0

        for i in range(1, len(h)):
            while h[i] < h[stack[-1]]:
                height = h[stack.pop()]	# 注意顺序
                left = stack[-1]          # 弹出后新的栈顶就是左边界
                width = i - left - 1
                ans = max(ans, height * width)
            stack.append(i)# 别忘每次加进去i

        return ans
```

------

关键细节（常见坑）

1. **栈存索引，不存高度**（需要用索引算宽度）
2. 弹栈时：
   - `height = h[pop_idx]`
   - `left = stack[-1]`（弹完后的栈顶是左边第一个更小）
   - `right = i`（当前 i 是右边第一个更小）
3. 为啥要哨兵 `0`？
   - 防止栈空
   - 强制最后把所有柱子都弹出计算

## [215. 数组中的第K个最大元素](https://leetcode.cn/problems/kth-largest-element-in-an-array/)

小顶堆（O(n log k)，最稳）

维护一个大小为 k 的小顶堆：

- 先放入前 k 个
- 后面遇到更大的就替换堆顶
- 最终堆顶就是第 k 大

```python
from typing import List
import heapq

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        heap = nums[:k]
        heapq.heapify(heap)

        for x in nums[k:]:
            if x > heap[0]:
                heapq.heapreplace(heap, x)# 注意语法弹出并返回 heap 的最小元素，然后将 x 推入堆中

        return heap[0]
```

## [347. 前 K 个高频元素](https://leetcode.cn/problems/top-k-frequent-elements/)

小顶堆 O(n log k)

思路：

- 维护一个大小为 k 的最小堆，堆里放 `(count, num)`
- 频次更大就替换堆顶
- 最后把堆中元素取出来

```python
from typing import List
from collections import Counter	# 导入的Counter在collections包里
import heapq

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        freq = Counter(nums)

        heap = []
        for num, c in freq.items():# 直接遍历哈希表（字典）
            if len(heap) < k:
                heapq.heappush(heap, (c, num))# 反向存 次数：数字
            else:
                if c > heap[0][0]:	# 注意是heap[0][0]
                    heapq.heapreplace(heap, (c, num))

        return [num for c, num in heap]
```

```python
from typing import List

class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        min_price = float('inf')
        ans = 0
        for p in prices:
            ans = max(ans, p - min_price)
            min_price = min(min_price, p)
        return ans
```

## [55. 跳跃游戏](https://leetcode.cn/problems/jump-game/)

思路（贪心）

维护目前能到达的最远位置 `far`：

- 从左到右遍历 `i`
- 如果 `i > far`：说明连 i 都到不了，直接 False
- 否则更新 `far = max(far, i + nums[i])`
- 最后只要 `far >= n-1` 就 True

```python
from typing import List

class Solution:
    def canJump(self, nums: List[int]) -> bool:
        far = 0
        for i, step in enumerate(nums):
            if i > far:# 在前
                return False
            far = max(far, i + step)# 在后
        return True
```

**贪心** 

维护三个变量：

- `end`：当前这一步能到达的最远边界（本层右边界）
- `far`：在遍历当前层时，下一步能到达的最远位置
- `steps`：跳跃次数

从左到右遍历到 `n-2`（最后一个不用跳）：

1. 更新 `far = max(far, i + nums[i])`
2. 当走到当前层边界 `i == end`，说明这一跳的覆盖范围用完了：
   - `steps += 1`
   - `end = far`（进入下一层）

```python
from typing import List

class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        steps = 0
        end = 0
        far = 0

        # 走到 n-2 即可：到达最后一个位置不需要再跳
        for i in range(n - 1):
            far = max(far, i + nums[i])
            if i == end:	# 当前这一步能到达的最远边界（本层右边界）
                steps += 1
                end = far

        return steps
```

- i=1: far=max(2,1+3)=4
- i=2: far=max(4,2+1)=4，i==end(2) → steps=2，end=4

## [763. 划分字母区间](https://leetcode.cn/problems/partition-labels/)

1. 先记录每个字符在字符串里 **最后一次出现的位置** `last[ch]`
2. 从左到右遍历，维护当前分段的右边界 `end = max(end, last[s[i]])`
3. 当 `i == end`，说明这一段里出现的所有字符的最后位置都在段内，**可以切一刀**：
   - 段长度 `end - start + 1`
   - 更新 `start = i + 1`

```python
from typing import List

class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        last = {c: i for i, c in enumerate(s)}  # 每个字符最后出现的位置

        res = []
        start = 0
        end = 0

        for i, c in enumerate(s):
            end = max(end, last[c])	# 这一段里出现的所有字符的最后位置
            if i == end:
                res.append(end - start + 1)
                start = i + 1 # 注意移动start指针

        return res
```


- 第二段到 `end=15` → 长度 7（"defegde"）
- 第三段到 `end=23` → 长度 8（"hijhklij"）

输出：`[9, 7, 8]`

## [31. 下一个排列](https://leetcode.cn/problems/next-permutation/)

**Trick**

核心思路（从右往左）

1. **找“下降点”**：从右往左找到第一个 `nums[i] < nums[i+1]` 的位置 `i`
   - 如果找不到，说明整个序列非递增（如 `[3,2,1]`），下一个排列就是最小排列：**整体反转**
2. **找右侧刚好比 nums[i] 大的元素**：从右往左找第一个 `nums[j] > nums[i]`
3. **交换 i 和 j**，然后把 `i+1..end` **反转**（让后缀变成最小）

```python
from typing import List

class Solution:
    def nextPermutation(self, nums: List[int]) -> None:
        n = len(nums)
        i = n - 2

        # 1) 找下降点 i
        while i >= 0 and nums[i] >= nums[i + 1]:
            i -= 1

        # 如果不存在下降点，直接反转全数组
        if i < 0:
            nums.reverse()
            return

        # 2) 找右侧第一个 > nums[i] 的 j
        j = n - 1
        while nums[j] <= nums[i]:
            j -= 1

        # 3) 交换
        nums[i], nums[j] = nums[j], nums[i]

        # 4) 反转后缀，i + 1 到最后
        l, r = i + 1, n - 1
        while l < r:
            nums[l], nums[r] = nums[r], nums[l]
            l += 1
            r -= 1
```

## [279. 完全平方数](https://leetcode.cn/problems/perfect-squares/)

代码随想录

定义 `dp[i]`：凑出和为 `i` 的最少平方数个数。

用 j 遍历完全平方数

dp[i] = min(dp[i], dp[i - j * j] + 1)

```python
from typing import List
import math

class Solution:
    def numSquares(self, n: int) -> int:
        dp = [0] + [10**9] * n # 不用这么大
        for i in range(1, n + 1):
            j = 1
            while j * j <= i:# j 从 1 到根号i
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1
        return dp[n]
```

**复杂度**：O(n * sqrt(n))，空间 O(n)

## [322. 零钱兑换](https://leetcode.cn/problems/coin-change/)

定义：`dp[j]` = 凑出金额 `j` 的最少硬币数
 初始化：

- 其他为无穷大 `INF`

```python
from typing import List

class Solution:
    def coinChange(self, coins: List[int], amount: int) -> int:
        INF = 10**9
        dp = [INF] * (amount + 1)
        dp[0] = 0

        for coin in coins:                     # 枚举硬币
            for j in range(coin, amount + 1):  # 正向遍历 => 可重复使用 coin
                dp[j] = min(dp[j], dp[j - coin] + 1)

        return -1 if dp[amount] == INF else dp[amount]
```

## [139. 单词拆分](https://leetcode.cn/problems/word-break/)

代码随想录

**如果求组合数就是外层for循环遍历物品，内层for遍历背包**。

**如果求排列数就是外层for遍历背包，内层for循环遍历物品**。

总结：求组合数：[动态规划：518.零钱兑换II (opens new window)](https://programmercarl.com/0518.零钱兑换II.html)求排列数：[动态规划：377. 组合总和 Ⅳ (opens new window)](https://programmercarl.com/0377.组合总和.html)、[动态规划：70. 爬楼梯进阶版（完全背包） (opens new window)](https://programmercarl.com/0070.爬楼梯完全背包版本.html)求最小数：[动态规划：322. 零钱兑换 (opens new window)](https://programmercarl.com/0322.零钱兑换.html)、[动态规划：279.完全平方数](https://programmercarl.com/0279.完全平方数.html)

`dp[i] = True/False`：表示 `s[0:i]`（前 i 个字符）能否被拆分
`dp[0] = True`：空串可拆

如果存在 `j < i`：

- `dp[j] == True` 且 `s[j:i] in wordDict`
  那么 `dp[i] = True`

```python
from typing import List

class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)
        n = len(s)
        max_len = max((len(w) for w in word_set), default=0)

        dp = [False] * (n + 1)
        dp[0] = True

        for i in range(1, n + 1): # 外层背包（容量）
            start = max(0, i - max_len)  # 只枚举可能的长度,最长物品
            for j in range(start, i):# 内层物品
                if dp[j] and s[j:i] in word_set:
                    dp[i] = True
                    break # 防止覆盖

        return dp[n]
```

```
s = "leetcode"`, `wordDict = ["leet","code"]
```

- `dp[0]=T`
- `i=4` 时：`s[0:4]="leet"` 在字典且 `dp[0]=T` → `dp[4]=T`
- `i=8` 时：`s[4:8]="code"` 在字典且 `dp[4]=T` → `dp[8]=T`
  最终 `dp[8]=True`。

1. **wordDict 一定要转 set**，否则查找会慢很多。
2. `dp[i]` 表示的是 **前 i 个字符**（注意切片 `s[j:i]` 右端是 i）。
3. 一旦 `dp[i]=True` 就可以 `break`，别继续浪费时间。
4. 大数据下加 `max_len` 很关键：不需要枚举所有 `j`。

`dp` 这版写法是 **标准的 Word Break（LeetCode 139）一维 DP**，它之所以正确，

核心在于：**dp[i] 表示 s 的前 i 个字符（s[0:i]）能否被拆分成字典单词**。

------

1) 先回答你问的边界：`s[j:i]` 包不包括右边界？

**不包括。**

在 Python 里切片规则是：

- `s[l:r]` 取的是 **下标 l 到 r-1**
- **左闭右开**：包含 `l`，不包含 `r`

所以 `s[j:i]` 代表的子串是：`s[j] ... s[i-1]`

举例：`s="abcdef"`

- `s[1:4] == "bcd"`（取 1,2,3，不取 4）

------

2) 为什么这段 DP 是对的？

定义（不变式）

`dp[i] = True` ⇔ `s[0:i]` 可以被拆成若干个字典单词。

- `dp[0]=True`：空字符串可以认为“已经成功拆分”（作为起点非常关键）

状态转移为什么成立？

对于每个 `i`，我们枚举一个断点 `j`：

如果：

1. `dp[j] == True` （说明前缀 `s[0:j]` 可拆分）
2. `s[j:i] in word_set`（说明后面这一段 `s[j:i]` 是一个字典单词）

那么就能把 `s[0:i]` 拆成：

```
(s[0:j]) + (s[j:i])
```

因此可以推出：`dp[i] = True`

这就是代码里的：

```
if dp[j] and s[j:i] in word_set:
    dp[i] = True
    break
```

`break` 是因为：**只要找到一种拆分方式就够了**（题目要的是 bool，不需要方案数/方案内容）。

------

3) 为什么 `start = max(0, i - max_len)` 不会漏？

`max_len` 是字典里单词最大长度。

如果 `s[j:i]` 是一个字典单词，那么它的长度一定满足：

```
i - j <= max_len`
 ⇒ `j >= i - max_len
```

所以你只需要枚举 `j` 从 `i-max_len` 到 `i-1`，更早的 `j` 产生的子串长度会超过最大单词长度，**不可能在字典里**，枚举了也白费。

因此这句只是优化，不影响正确性。

------

4) 用一个小例子走一遍（直觉更强）

```
s = "leetcode"`, `wordDict = ["leet", "code"]
```

- `dp[0]=True`
- `i=4` 时，检查 `j=0`：
   `dp[0]=True` 且 `s[0:4]="leet"` 在字典
   ⇒ `dp[4]=True`
- `i=8` 时，检查 `j=4`：
   `dp[4]=True` 且 `s[4:8]="code"` 在字典
   ⇒ `dp[8]=True`
   最终 `dp[n]=dp[8]=True`

LeetCode **152：乘积最大子数组** 的关键点是：**负数会把最大/最小互换**，所以要同时维护：

- `max_end`：以当前位置结尾的最大乘积
- `min_end`：以当前位置结尾的最小乘积（因为乘以负数可能变最大）

转移（遍历每个 `x`）：

- 新的 `max_end = max(x, x*max_end, x*min_end)`
- 新的 `min_end = min(x, x*max_end_old, x*min_end_old)`
- 全局答案取 `max(ans, max_end)`

```python
from typing import List

class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        max_end = min_end = ans = nums[0]

        for x in nums[1:]:
            # 先保存旧值，避免被覆盖
            prev_max, prev_min = max_end, min_end

            max_end = max(x, x * prev_max, x * prev_min)
            min_end = min(x, x * prev_max, x * prev_min)

            ans = max(ans, max_end)

        return ans
```

例子 `[-2, 3, -4]`：

- 到 3：最大是 3，最小是 -6
- 遇到 -4：`(-4) * (-6) = 24` 反而变成最大，所以必须保留“最小（最负）”来翻盘。

## [22. 括号生成](https://leetcode.cn/problems/generate-parentheses/)

1. **合法括号的特性**：
   - 每一对括号必须成对出现，并且**左括号必须在右括号之前**。
   - 如果我们已经放了 `n` 个左括号，不能再放左括号；如果已经放了 `n` 个右括号，不能再放右括号。
2. **回溯法框架**：
   - 设定递归函数 `backtrack`，参数包括当前的括号字符串、已使用的左括号数量 `left` 和已使用的右括号数量 `right`。
   - 基本情况：
     - 当左括号数量等于右括号数量等于 `n` 时，意味着括号组合已经完成，将其加入结果列表。
   - 递归：
     - 如果左括号数量小于 `n`，可以再添加一个左括号。
     - 如果右括号数量小于左括号数量，可以再添加一个右括号。

```python
class Solution:
    def generateParenthesis(self, n: int):
        result = []

        def backtrack(current_string, left, right):
            # 基本情况：如果当前字符串长度等于 2 * n，表示构建了一个有效的括号组合
            if len(current_string) == 2 * n:
                result.append(current_string)
                return

            # 尝试添加左括号
            if left < n:
                backtrack(current_string + '(', left + 1, right)

            # 尝试添加右括号
            if right < left:
                backtrack(current_string + ')', left, right + 1)

        # 启动回溯函数，初始时括号字符串为空，左括号和右括号的数量均为 0
        backtrack('', 0, 0)
        return result
```

## [79. 单词搜索](https://leetcode.cn/problems/word-search/)

1. **回溯法的应用**：
   - 从每个单元格开始，尝试在四个方向（上、下、左、右）查找单词。
   - 一旦找到单词的某一部分，继续从下一个字符开始查找，直到整个单词都被找到。
   - 每个单元格只能使用一次，因此需要**在每次回溯前标记已访问的单元格，回溯时再恢复标记**。
2. **回溯法的步骤**：
   - 遍历每一个单元格，尝试将其作为起点。
   - 对于每个字符，检查它是否和当前字符匹配，如果匹配，则继续在周围的四个方向进行搜索。
   - 如果搜索到字符串的末尾，说明找到了该单词，返回 `True`。
   - 如果无法继续搜索，返回 `False`。

```python
class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        def backtrack(i, j, index):
            # 边界条件：如果超出边界或字符不匹配
            if i < 0 or i >= len(board) or j < 0 or j >= len(board[0]) or board[i][j] != word[index]:
                return False
            
            # 如果已经找到整个单词
            if index == len(word) - 1:
                return True
            
            # 暂时标记当前位置，避免重复使用
            temp = board[i][j]
            board[i][j] = '#'
            
            # 在四个方向上进行回溯
            result = (backtrack(i + 1, j, index + 1) or  # 向下
                      backtrack(i - 1, j, index + 1) or  # 向上
                      backtrack(i, j + 1, index + 1) or  # 向右
                      backtrack(i, j - 1, index + 1))    # 向左
            
            # 恢复当前位置的值
            board[i][j] = temp
            
            return result
        
        # 遍历每个单元格作为起点
        for i in range(len(board)):
            for j in range(len(board[0])):
                # 从每个起点开始回溯
                if backtrack(i, j, 0):
                    return True
        
        return False
```

**`backtrack(i, j, index)`**：

- `i` 和 `j` 表示当前的单元格位置。
- `index` 表示我们正在寻找的单词的字符索引。
- 首先检查当前位置是否越界，或者该位置的字符是否与当前的字符不匹配。
- 如果匹配，继续往四个方向进行回溯（上下左右）。
- 如果当前字符是单词的最后一个字符，说明找到了整个单词，返回 `True`。
- 如果没有找到，恢复当前位置的字符，并返回 `False`。

```python
class Solution:
    def partition(self, s: str) -> List[List[str]]:
        result = []
        self.backtracking(s, 0, [], result)
        return result
    def backtracking(self, s, start_index, path, result ):
        if start_index == len(s):
            result.append(path[:])
            return
        for i in range(start_index, len(s)):
            # 此次比其他组合题目多了一步判断：
            # 判断被截取的这一段子串([start_index, i])是否为回文串
            if self.is_palindrome(s, start_index, i):
                path.append(s[start_index:i+1])
                self.backtracking(s, i+1, path, result)
                path.pop()
    def is_palindrome(self, s: str, start: int, end: int) -> bool:
        i: int = start        
        j: int = end
        while i < j:
            if s[i] != s[j]:
                return False
            i += 1
            j -= 1
        return True 
```

## [208. 实现 Trie (前缀树)](https://leetcode.cn/problems/implement-trie-prefix-tree/)

Trie 的每个节点包含：

- `children`：从字符到子节点的映射（常用 dict；如果只含小写字母也可用长度 26 的数组）
- `is_end`：是否是某个单词的结尾

操作：

- `insert(word)`：沿路径创建节点，最后标记 `is_end=True`
- `search(word)`：沿路径走，最后要求 `is_end=True`
- `startsWith(prefix)`：沿路径走，只要能走完就 True（不要求 `is_end`）

```python
class TrieNode:
    def __init__(self):
        self.children = {}   # 字符 -> 子节点 的映射，比如 {'a': TrieNode, 'b': TrieNode}
        self.is_end = False  # 标记：从根走到当前节点是否构成了一个“完整单词”的结尾

class Trie:
    def __init__(self):
        self.root = TrieNode()  # 根节点，不对应具体字符（空节点）
    def insert(self, word: str) -> None:
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word: str) -> bool:
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

    def startsWith(self, prefix: str) -> bool:
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True
```

保持两个不变量：

1. `len(small)` == `len(large)` 或 `len(small)` == `len(large) + 1`
    （让左边最多比右边多 1 个）
2. `max(left) <= min(right)`
    即 `-small[0] <= large[0]`（如果两边都非空）

这样：

- 如果总数是奇数：中位数 = `-small[0]`
- 如果总数是偶数：中位数 = (`-small[0] + large[0]`) / 2

```python
import heapq

class MedianFinder:
    def __init__(self):
        self.small = []  # max-heap via negatives
        self.large = []  # min-heap

    def addNum(self, num: int) -> None:
        # 1) 先放到 small（左边）
        heapq.heappush(self.small, -num)

        # 2) 保证 small 的最大值 <= large 的最小值
        #    把 small 最大的挪到 large
        heapq.heappush(self.large, -heapq.heappop(self.small))# 大的放过去

        # 3) 平衡数量：让 small 至少不比 large 少
        if len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))# 小的放过来

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return float(-self.small[0])
        return (-self.small[0] + self.large[0]) / 2.0
```

- `large = []`

- large: `[]`

- small: `[]`
- large: `[1]`

Step3：平衡数量（large 比 small 多）

- pop large 的最小值 1 放回 small
- small: `[1]`
- large: `[]`

当前数据：`[1]`（奇数）

- median = `max(small)=1`

## [32. 最长有效括号](https://leetcode.cn/problems/longest-valid-parentheses/)

**DP（dp[i] 表示“以 i 结尾的最长有效长度”）**

定义：`dp[i]` = 以 `s[i]` 结尾的最长有效括号长度（所以 `s[i]` 必须是 `')'` 才可能 >0）

转移：

1. `...()` 结尾：如果 `s[i]==')'` 且 `s[i-1]=='('`
    `dp[i] = dp[i-2] + 2`
2. `...))` 结尾：如果 `s[i]==')'` 且 `s[i-1]==')'`
    设 `pre = i - dp[i-1] - 1`（i 前面与当前 ')' 可能匹配的 '(' 位置）
    若 `pre >= 0` 且 `s[pre]=='('`：
    `dp[i] = dp[i-1] + 2 + dp[pre-1]`（把前面接上）

```python
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        n = len(s)
        dp = [0] * n
        ans = 0

        for i in range(1, n):
            if s[i] == ')':
                if s[i - 1] == '(':
                    dp[i] = (dp[i - 2] if i >= 2 else 0) + 2
                else:
                    pre = i - dp[i - 1] - 1
                    if pre >= 0 and s[pre] == '(':
                        dp[i] = dp[i - 1] + 2 + (dp[pre - 1] if pre >= 1 else 0)
                ans = max(ans, dp[i])

        return ans
```

## [155. 最小栈](https://leetcode.cn/problems/min-stack/)--40

**双栈（推荐）**

- `stack`: 正常存元素
- `min_stack`: 同步存“到当前位置为止的最小值”
  - 每次 push：把 `min(当前值, min_stack[-1])` 压入 `min_stack`
  - 每次 pop：两边一起 pop
     这样 `getMin()` 直接返回 `min_stack[-1]`。

```python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val: int) -> None:
        self.stack.append(val)
        if not self.min_stack:
            self.min_stack.append(val)
        else:
            self.min_stack.append(min(val, self.min_stack[-1]))

    def pop(self) -> None:
        self.stack.pop()
        self.min_stack.pop()

    def top(self) -> int:
        return self.stack[-1]

    def getMin(self) -> int:
        return self.min_stack[-1]
```

## [146. LRU 缓存](https://leetcode.cn/problems/lru-cache/)

有详细题解

```python
class DLinkedNode:
    def __init__(self, key = 0, value = 0):# 注意要给初始值

        self.key = key
        self.value =value
        self.prev = None
        self.next = None

class LRUCache:

    def __init__(self, capacity: int):
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        self.head.next = self.tail
        self.tail.prev = self.head
        self.capacity = capacity
        self.cache = {}
        self.size = 0

    # 删除节点
    def _remove(self, node: DLinkedNode):
        node.prev.next = node.next
        node.next.prev = node.prev

    # 增加节点到头
    def _add_to_head(self, node: DLinkedNode):

        node.next = self.head.next
        self.head.next.prev = node
        self.head.next = node
        node.prev = self.head

    # 节点移到头部
    def _move_to_head(self, node):
        # 先删后增
        self._remove(node)
        self._add_to_head(node)

    # 删除链表尾部节点
    def _remove_tail(self):
        node = self.tail.prev
        self._remove(node)
        return node


    def get(self, key: int) -> int:
        # 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
        if key in self.cache:
            node = self.cache[key]
            self._move_to_head(node)
            return node.value
        else:
            return -1

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self._move_to_head(node)
        else:
            node = DLinkedNode(key, value)
            self.cache[key] = node
            self.size += 1
            self._add_to_head(node)
            if self.size > self.capacity:
                removed = self._remove_tail()
                del self.cache[removed.key]# 注意 删掉尾部节点的缓存记录
                self.size -= 1
```

## [1. 两数之和](https://leetcode.cn/problems/two-sum/)

题目要找 **两个数的下标** `i, j`，使得 `nums[i] + nums[j] = target`，并且同一个元素不能用两次。

最经典做法：**哈希表（字典）记录“数值 → 下标”**，一边遍历一边查：

- 遍历到 `x = nums[i]`，希望另一个数是 `need = target - x`
- 如果 `need` 已经在哈希表里，说明之前出现过 `need`，直接返回 `[哈希表[need], i]`
- 否则，把当前 `x` 和它的下标 `i` 放进哈希表，继续遍历

```python
from typing import List

class Solution:
    def twoSum(self, nums: List[int], target: int) -> List[int]:
        # 哈希表：记录已经遍历过的数 -> 它的下标
        idx = {}
        for i, x in enumerate(nums):
            need = target - x  # 需要的另一个数
            if need in idx:
                # 找到了：need 的下标在 idx[need]，当前数的下标是 i
                return [idx[need], i]
            # 没找到就记录当前数，供后面使用
            idx[x] = i
        return []
```

## [128. 最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/)

要找数组中**最长的连续整数序列长度**（不要求原数组顺序连续），经典解法是 **哈希集合 set + 只从“序列起点”开始扩展**。

关键观察：

- 如果一个数 `x` 是某个连续序列的**起点**，那么 `x-1` **不在**集合里。
- 只有从起点开始往右扩展 `x, x+1, x+2...` 才不会重复做无用功。
- 每个数最多被“扩展扫描”一次，总体是 `O(n)`。

```python
from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        s = set(nums)  # 用set去重 + O(1) 查找
        ans = 0

        for x in s:#　避免对已经处理过的数字重复计算，可以在遍历过程中标记每个数字，防止重复计算。
            # 只有当 x 是“序列起点”才开始扩展
            if x - 1 not in s:
                cur = x
                length = 1

                # 不断往右找连续数
                while cur + 1 in s:
                    cur += 1
                    length += 1

                ans = max(ans, length)

        return ans
```

```
nums = [100, 4, 200, 1, 3, 2]`
set = {100, 4, 200, 1, 3, 2}
```

遍历：

- `1`：`0` 不在 set → 是起点，扩展 `1,2,3,4` 长度 4 → ans=4
- `2`：`1` 在 set → 不是起点（跳过）
- `3`：`2` 在 set → 跳过
- `4`：`3` 在 set → 跳过
- `100`：`99` 不在 → 长度 1
- `200`：`199` 不在 → 长度 1

最终答案：4

## [11. 盛最多水的容器](https://leetcode.cn/problems/container-with-most-water/)

要在数组 `height` 里选两根柱子 `i, j` 作为容器边界，面积：

$$
area=(j−i)×min⁡(height[i],height[j])
$$
暴力枚举 `O(n^2)` 太慢。最优解是 **双指针**：

- 左指针 `l` 从 0 开始，右指针 `r` 从 n-1 开始
- 每次计算当前面积，更新答案
- **移动较短的那一侧**：
  - 如果 `height[l] < height[r]`，移动 `l += 1`
  - 否则移动 `r -= 1`

为什么移动短板？

- 宽度 `(r-l)` 变小是必然的，想让面积变大只能指望 `min(height[l], height[r])` 变大
- 当前短板限制了高度，不移动短板而移动长板，`min` 不可能变大（短板还在），面积只会更差或不变
- 所以只移动短板才可能找到更优解

```python
from typing import List

class Solution:
    def maxArea(self, height: List[int]) -> int:
        l, r = 0, len(height) - 1
        ans = 0

        while l < r:
            # 当前面积
            h = min(height[l], height[r])
            ans = max(ans, (r - l) * h)

            # 移动短板，才可能提高 min(height[l], height[r])
            if height[l] < height[r]:
                l += 1
            else:
                r -= 1

        return ans
```

```
height = [1,8,6,2,5,4,8,3,7]
```

- l=0(1), r=8(7) 面积=8*1=8 → 短板在 l，l++
- l=1(8), r=8(7) 面积=7*7=49 → 更新 ans=49 → 短板在 r，r--
- l=1(8), r=7(3) 面积=6*3=18 → r--
- …继续直到 l>=r，最终答案 49

## [15. 三数之和](https://leetcode.cn/problems/3sum/)

核心思路（**排序** + 双指针）

要找所有三元组 `(a,b,c)` 使得 `a+b+c=0`，且结果**不重复**。

经典做法：

1. **先排序** `nums.sort()`
2. 枚举第一个数的下标 `i`（固定 `nums[i]`）
3. 在 `i` 右侧用 **双指针**：
   - `l = i+1`, `r = n-1`
   - 计算 `s = nums[i] + nums[l] + nums[r]`
     - `s == 0`：找到一组，收集后同时移动 `l`、`r`，并跳过重复值
     - `s < 0`：说明和太小，需要更大 → `l += 1`
     - `s > 0`：说明和太大，需要更小 → `r -= 1`

**去重关键：**

- 枚举 `i` 时：如果 `nums[i] == nums[i-1]`，跳过（避免同一首元素重复）
- 找到答案后：移动 `l` 时跳过所有相同的 `nums[l]`；移动 `r` 时跳过所有相同的 `nums[r]`

```python
from typing import List

class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        res = []

        for i in range(n):
            # 1) i 去重：同一个 nums[i] 只处理一次
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            # 可选剪枝：最小都 > 0，后面不可能凑出 0
            if nums[i] > 0:
                break

            l, r = i + 1, n - 1
            while l < r:
                s = nums[i] + nums[l] + nums[r]
                if s == 0:
                    res.append([nums[i], nums[l], nums[r]])

                    # 2) l 去重
                    l += 1
                    while l < r and nums[l] == nums[l - 1]:
                        l += 1

                    # 3) r 去重
                    r -= 1
                    while l < r and nums[r] == nums[r + 1]:
                        r -= 1

                elif s < 0:
                    l += 1
                else:
                    r -= 1

        return res
```

```
nums = [-1, 0, 1, 2, -1, -4]`
排序后：`[-4, -1, -1, 0, 1, 2]
```

- i=0, nums[i]=-4：双指针怎么配都不够 0（最终无解）
- i=1, nums[i]=-1：
  - l=2(-1), r=5(2) → sum=0  得到 `[-1,-1,2]`
  - l=3(0), r=4(1) → sum=0  得到 `[-1,0,1]`
- i=2 时 nums[i] 还是 -1，和 i=1 重复 → 跳过
  最终：`[[-1,-1,2], [-1,0,1]]`

## [3. 无重复字符的最长子串](https://leetcode.cn/problems/longest-substring-without-repeating-characters/)

核心思路（滑动窗口 + **哈希表**）

要找最长子串，使得子串内**没有重复字符**。经典做法是滑动窗口：

- 用两个指针 `l, r` 表示当前窗口 `[l, r]`
- 用哈希表 `last` 记录每个字符**最近一次出现的位置**
- 当遍历到 `s[r] = ch`：
  - 如果 `ch` 在窗口内出现过（即 `last[ch] >= l`），说明重复了→ 把左边界 `l` 直接跳到 `last[ch] + 1`
  - 更新 `last[ch] = r`
  - 更新答案 `ans = max(ans, r - l + 1)`

关键点：`l` 只能向右移动，且一次跳跃到位，整体 `O(n)`。

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        last = {}   # 记录字符最近一次出现的下标
        l = 0       # 滑动窗口左边界
        ans = 0
        for r, ch in enumerate(s):
            # 如果 ch 在当前窗口 [l, r-1] 内出现过，移动左边界
            if ch in last and last[ch] >= l:
                l = last[ch] + 1
            # 更新 ch 最近出现位置
            last[ch] = r
            # 更新最大长度
            ans = max(ans, r - l + 1)
        return ans
```

------

示例演示：`s = "pwwkew"`

我们跟踪 `l`、`r`、窗口内容、答案：

- r=0, 'p'：没重复，窗口 `"p"`，ans=1
- r=1, 'w'：没重复，窗口 `"pw"`，ans=2
- r=2, 'w'：重复（上次 'w' 在 1，且 1>=l=0）
  → l=1+1=2，窗口变 `"w"`，ans 仍 2
- r=3, 'k'：窗口 `"wk"`，ans=2
- r=4, 'e'：窗口 `"wke"`，ans=3
- r=5, 'w'：重复（上次 'w' 在 2，且 2>=l=2）→ l=2+1=3，窗口 `"kew"`，ans 仍 3

最终答案：3（"wke" 或 "kew"）

## [438. 找到字符串中所有字母异位词](https://leetcode.cn/problems/find-all-anagrams-in-a-string/)--50

要在 `s` 中找所有长度为 `len(p)` 的子串，使它与 `p` **互为字母异位词**。

异位词判定本质：**26 个字母频次相同**。

做法：固定窗口长度 `m = len(p)`，在 `s` 上滑动窗口 `[l, r]`（始终保持长度 m）：

- 先统计 `p` 的计数 `need[26]`
- 维护窗口计数 `win[26]`
- 每次右移加入一个字符、如果窗口超长就移出左端字符
- 当窗口长度恰好为 `m` 时，若 `win == need`，则 `l` 是一个答案起点

```python
from typing import List

class Solution:
    def findAnagrams(self, s: str, p: str) -> List[int]:
        n, m = len(s), len(p)
        if m > n:
            return []

        need = [0] * 26
        win = [0] * 26

        # 统计 p 的频次
        for ch in p:
            need[ord(ch) - ord('a')] += 1

        res = []
        l = 0

        for r, ch in enumerate(s):
            # 1) 扩展右边界：加入 s[r]
            win[ord(ch) - ord('a')] += 1

            # 2) 如果窗口长度超过 m，则收缩左边界
            if r - l + 1 > m:
                win[ord(s[l]) - ord('a')] -= 1
                l += 1

            # 3) 窗口长度等于 m 时判断是否异位词
            if r - l + 1 == m and win == need:
                res.append(l)

        return res
```

- `"bae"` ✗
- `"aeb"` ✗
- `"eba"` ✗
- `"bab"` ✗
- `"aba"` ✗
- `"bac"` ✅ → 起点 6
- `"acd"` ✗

## [53. 最大子数组和](https://leetcode.cn/problems/maximum-subarray/)

**动态规划**

设 `dp[i]` 表示**以 `i` 结尾**的最大子数组和，那么：

- 要么把 `nums[i]` 接在前面子数组后面：`dp[i-1] + nums[i]`
- 要么从 `nums[i]` 重新开始：`nums[i]`

所以转移：
$$
dp[i] = \max(nums[i],\ dp[i-1] + nums[i])
$$
答案是所有 `dp[i]` 的最大值。

为了省空间，用一个变量 `cur` 表示当前的 `dp`，`ans` 表示全局最大值。

```python
from typing import List

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        # cur：以当前位置结尾的最大子数组和
        cur = nums[0]
        # ans：全局最大子数组和
        ans = nums[0]

        for i in range(1, len(nums)):
            # 要么延续前面的子数组，要么从 nums[i] 重新开始
            cur = max(nums[i], cur + nums[i])
            ans = max(ans, cur)

        return ans
```

```
nums = [-2,1,-3,4,-1,2,1,-5,4]
```

- i=0：cur=-2, ans=-2
- i=1：cur=max(1, -2+1=-1)=1，ans=1
- i=2：cur=max(-3, 1-3=-2)=-2，ans=1

## [56. 合并区间](https://leetcode.cn/problems/merge-intervals/)

核心思路（排序 + 扫描合并）

区间合并的关键：**先按起点排序**，再从左到右扫描。

步骤：

1. 将所有区间按 `start` 升序排序（若 `start` 相同按 `end` 升序也行）
2. 维护结果数组 `res`
3. 对每个区间 `[s, e]`：
   - 如果 `res` 为空，或 `s > res[-1][1]`（与上一个区间不重叠）
     → 直接加入 `res`
   - 否则重叠（`s <= res[-1][1]`）
     → 合并：更新 `res[-1][1] = max(res[-1][1], e)`

```python
from typing import List

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 1) 按区间起点排序
        intervals.sort(key=lambda x: x[0])

        res = []
        for s, e in intervals:
            # 2) 如果 res 为空，或当前区间与最后一个区间不重叠：直接加入
            if not res or s > res[-1][1]:
                res.append([s, e])
            else:
                # 3) 重叠：合并到最后一个区间
                res[-1][1] = max(res[-1][1], e)

        return res
```

输入：`intervals = [[1,3],[2,6],[8,10],[15,18]]`

排序后仍是：
`[[1,3],[2,6],[8,10],[15,18]]`

扫描：

- res=[] → 加入 [1,3]
- 当前 [2,6]：2 <= 3 重叠 → 合并成 [1,6]
- 当前 [8,10]：8 > 6 不重叠 → 加入 [8,10]
- 当前 [15,18]：15 > 10 不重叠 → 加入 [15,18]

输出：`[[1,6],[8,10],[15,18]]`

## [189. 轮转数组](https://leetcode.cn/problems/rotate-array/)

核心思路（原地：三次翻转）

把数组向右轮转 `k` 位：最后 `k` 个元素移动到最前面。

关键：`k` 可能大于 `n`，先做 `k %= n`。

**三次翻转法**（最经典、O(1) 空间）：

1. 翻转整个数组
2. 翻转前 `k` 个
3. 翻转后 `n-k` 个

为什么有效？

- 右轮转相当于把数组拆成 `A | B`（`B` 长度为 k），结果应为 `B | A`
- 全翻转后变成 `rev(B) | rev(A)`
- 再分别翻转两段就变回 `B | A`

```python
from typing import List

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        n = len(nums)
        if n == 0:
            return
        k %= n
        if k == 0:
            return

        # 原地翻转 nums[l..r]
        def reverse(l: int, r: int) -> None:
            while l < r:
                nums[l], nums[r] = nums[r], nums[l]
                l += 1
                r -= 1

        # 1) 翻转整个数组
        reverse(0, n - 1)
        # 2) 翻转前 k 个
        reverse(0, k - 1)
        # 3) 翻转后 n-k 个
        reverse(k, n - 1)
```

```
nums = [1,2,3,4,5,6,7], k = 3
```

- `k %= 7 => 3`

1. 全翻转：`[7,6,5,4,3,2,1]`
2. 翻转前 3 个：`[5,6,7,4,3,2,1]`
3. 翻转后 4 个：`[5,6,7,1,2,3,4]`

结果就是右移 3 位 

## [238. 除了自身以外数组的乘积](https://leetcode.cn/problems/product-of-array-except-self/)

核心思路（前缀积 + 后缀积，O(1) 额外空间）

要求 `ans[i] = nums[0]*...*nums[i-1] * nums[i+1]*...*nums[n-1]`，且不能用除法。

做法：

1. 先从左到右，算 **前缀积**：
   - `ans[i]` 先存 `i` 左边所有数的乘积
2. 再从右到左，用一个变量 `right` 维护 **后缀积**：
   - `right` 表示当前位置 `i` 右边所有数的乘积
   - 最终 `ans[i] *= right`

这样只用输出数组 `ans`，额外变量常数级。

```python
from typing import List

class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        ans = [1] * n
# 遍历时就保存结果->能少遍历一次
        # 1) 前缀积：ans[i] = nums[0] * ... * nums[i-1]
        left = 1
        for i in range(n):
            ans[i] = left
            left *= nums[i]

        # 2) 后缀积：right = nums[i+1] * ... * nums[n-1]
        right = 1
        for i in range(n - 1, -1, -1):
            ans[i] *= right
            right *= nums[i]
        return ans
```

## [41. 缺失的第一个正数](https://leetcode.cn/problems/first-missing-positive/)

核心思路（**原地哈希 / 下标置换**）

要找**最小缺失的正整数**，并且要求 `O(n)` 时间、`O(1)` 额外空间。

关键观察：
对长度为 `n` 的数组，答案一定在 `[1, n+1]` 之间。

我们可以把数组当作“哈希表”用：
**把值 `x`（若 `1 <= x <= n`）放到下标 `x-1` 的位置上**。
也就是说，理想状态应该是：

- `nums[0] == 1`
- `nums[1] == 2`
- ...
- `nums[n-1] == n`

```python
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)

        # 1) 原地把 1..n 放到对应位置：nums[x-1] = x
        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                j = nums[i] - 1
                nums[i], nums[j] = nums[j], nums[i]

        # 2) 找第一个不匹配的位置
        for i in range(n):
            if nums[i] != i + 1:
                return i + 1

        # 3) 都匹配，说明是 n+1
        return n + 1
```

`nums = [3, 4, -1, 1]`, n=4
目标：把 1 放到 idx0，2 放 idx1，3 放 idx2，4 放 idx3

经过交换可整理为：`[1, -1, 3, 4]`
扫描：

- idx0 是 1 ✅
- idx1 不是 2 ❌ → 答案 `2`

## [54. 螺旋矩阵](https://leetcode.cn/problems/spiral-matrix/)

**纯模拟**

核心思路（模拟边界收缩）

螺旋遍历就是沿着矩阵外圈走一圈，再进入内圈继续走。用四个边界控制：

- `top`：当前未遍历区域的上边界
- `bottom`：下边界
- `left`：左边界
- `right`：右边界

每一圈按顺序做 4 步：

1. 从左到右遍历 `top` 行，然后 `top += 1`
2. 从上到下遍历 `right` 列，然后 `right -= 1`
3. 若 `top <= bottom`，从右到左遍历 `bottom` 行，然后 `bottom -= 1`
4. 若 `left <= right`，从下到上遍历 `left` 列，然后 `left += 1`

注意第 3、4 步必须加判断，避免单行/单列重复遍历。

复杂度：

- 时间 `O(mn)`
- 空间 `O(1)`（不算输出数组）

```python
from typing import List

class Solution:
    def spiralOrder(self, matrix: List[List[int]]) -> List[int]:
        if not matrix or not matrix[0]:
            return []

        m, n = len(matrix), len(matrix[0])
        top, bottom = 0, m - 1
        left, right = 0, n - 1

        res = []

        while top <= bottom and left <= right:
            # 1) top 行：left -> right
            for j in range(left, right + 1):
                res.append(matrix[top][j])
            top += 1

            # 2) right 列：top -> bottom
            for i in range(top, bottom + 1):
                res.append(matrix[i][right])
            right -= 1

            # 3) bottom 行：right -> left（需要检查是否还有行）
            if top <= bottom:
                for j in range(right, left - 1, -1):
                    res.append(matrix[bottom][j])
                bottom -= 1

            # 4) left 列：bottom -> top（需要检查是否还有列）
            if left <= right:
                for i in range(bottom, top - 1, -1):
                    res.append(matrix[i][left])
                left += 1

        return res
```

输入：

```
matrix = [
  [1,  2,  3],
  [4,  5,  6],
  [7,  8,  9]
]
```

遍历顺序：

- 上行：1 2 3
- 右列：6 9
- 下行：8 7
- 左列：4
- 内圈：5

输出：
`[1,2,3,6,9,8,7,4,5]`

## [240. 搜索二维矩阵 II](https://leetcode.cn/problems/search-a-2d-matrix-ii/)

核心思路（从右上角“**走楼梯**”）

矩阵满足：

- 每行从左到右递增
- 每列从上到下递增

最经典解法：从 **右上角**（或左下角）开始搜索。

设当前位置 `(i, j)` 初始为右上角 `(0, n-1)`：

- 若 `matrix[i][j] == target`：找到
- 若 `matrix[i][j] > target`：说明这一列下面都更大，不可能是 target → **左移** `j -= 1`
- 若 `matrix[i][j] < target`：说明这一行左边都更小，不可能是 target → **下移** `i += 1`

每一步都能排除一整行或一整列，所以最多走 `m+n` 步.

```python
from typing import List

class Solution:
    def searchMatrix(self, matrix: List[List[int]], target: int) -> bool:
        if not matrix or not matrix[0]:
            return False

        m, n = len(matrix), len(matrix[0])
        i, j = 0, n - 1  # 从右上角开始

        while i < m and j >= 0:
            x = matrix[i][j]
            if x == target:
                return True
            elif x > target:
                # 当前值太大，排除当前列
                j -= 1
            else:
                # 当前值太小，排除当前行
                i += 1
        return False
```

## [160. 相交链表](https://leetcode.cn/problems/intersection-of-two-linked-lists/)

核心思路（双指针“走同样的路程”）

两条链表可能在某个节点开始相交（之后完全共享尾部）。要找相交起点节点。

经典做法：**两个指针分别从 A、B 出发，走到尾后切换到另一条链表头**：

- 指针 `pA` 从 `headA` 开始走
- 指针 `pB` 从 `headB` 开始走
- 当某指针走到 `None`，就切换到另一条链表头继续走
- 最终：
  - 如果相交，它们会在相交点相遇
  - 如果不相交，它们会同时走到 `None` 相遇（返回 `None`）

为什么一定会对齐？

- `pA` 走过的路径：A + B
- `pB` 走过的路径：B + A
  两者总路程一样，若有共同尾部，就会在进入共同尾部的同一时刻相遇。

```python
class Solution:
    def getIntersectionNode(self, headA: 'ListNode', headB: 'ListNode') -> 'ListNode':
        pA, pB = headA, headB

        # 当相交时会在相交点相遇；不相交时最终都为 None
        while pA is not pB:
            # 走到末尾就切换到另一条链表头
            pA = pA.next if pA else headB
            pB = pB.next if pB else headA

       return pA
```

假设：

- A：`1 -> 2 -> 3 -> 8 -> 9`
- B：`4 -> 5 -> 8 -> 9`
  其中 `8` 是相交起点（后面 `8 -> 9` 是共享的）

`pA` 走：A 走完到 `None`，切到 B
`pB` 走：B 走完到 `None`，切到 A
当两者都走了同样总长度后，会在 `8` 同步到达并相遇。

## [206. 反转链表](https://leetcode.cn/problems/reverse-linked-list/)

反转单链表本质：把每个节点的 `next` 指针方向反过来。

用三个指针：

- `prev`：已经反转好的部分的头（初始 `None`）
- `cur`：当前处理的节点（初始 `head`）
- `nxt`：暂存 `cur.next`（避免断链）

循环操作：

1. `nxt = cur.next` 暂存后继
2. `cur.next = prev` 反转指向
3. `prev = cur` 前进
4. `cur = nxt` 前进

最后 `prev` 就是新头结点。

```python
class Solution:
    def reverseList(self, head: 'ListNode') -> 'ListNode':
        prev = None
        cur = head

        while cur:
            nxt = cur.next    
            cur.next = prev     
            prev = cur         
            cur = nxt          

        return pre
```

## [234. 回文链表](https://leetcode.cn/problems/palindrome-linked-list/)

核心思路（**快慢指针 + 反转后半段**）

1. 用快慢指针找中点：`fast` 每次走 2 步，`slow` 每次走 1 步。
2. 若链表长度为奇数（`fast` 最后不为 `None`），需要**跳过中间那个节点**：`slow = slow.next`。
3. 反转 `slow` 开始的后半段。
4. 用两个指针从头和从后半段头同时走，逐个比较。

时间 `O(n)`，额外空间 `O(1)`。

```python
class Solution:
    def isPalindrome(self, head: 'ListNode') -> bool:
        if not head or not head.next:
            return True

        # 1) 快慢指针找中点
        slow, fast = head, head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

        # 易错点：奇数长度要跳过中间点
        if fast:
            slow = slow.next

        # 2) 反转后半段
        def reverse(node: 'ListNode') -> 'ListNode':
            prev, cur = None, node
            while cur:
                nxt = cur.next
                cur.next = prev  # 关键：反转指针
                prev = cur
                cur = nxt
            return prev

        second = reverse(slow)

        # 3) 对比两段（只需要遍历后半段长度）
        p1, p2 = head, second
        while p2:
            if p1.val != p2.val:
                return False
            p1 = p1.next
            p2 = p2.next

        return True
```

链表：`1 -> 2 -> 2 -> 1`

- 找中点后，`slow` 指向后半段起点
- 反转后半段得到 `1 -> 2`
- 逐个比较：`1==1`、`2==2` → 返回 `True`

## [142. 环形链表 II](https://leetcode.cn/problems/linked-list-cycle-ii/)

核心思路（**Floyd 快慢指针**：先判环，再找入口）

1. **判环**：`slow` 走 1 步、`fast` 走 2 步
   - 若相遇：有环
   - 若 `fast` 或 `fast.next` 为空：无环，返回 `None`
2. **找环入口**：相遇后令 `p1 = head`，`p2 = slow`（或 fast，反正它们在相遇点）
   - 两个指针都每次走 1 步
   - 再次相遇的位置就是**环的入口节点**

直观原因：设头到入口距离为 `a`，入口到相遇点在环上距离为 `b`，环长为 `c`。相遇时满足关系可推出：从头走 `a` 步与从相遇点走 `a` 步都会到入口，因此同步前进会在入口相遇。

```python
class Solution:
    def detectCycle(self, head: 'ListNode') -> 'ListNode':
        slow, fast = head, head

        # 易错点：必须同时检查 fast 和 fast.next，避免空指针
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:  # 关键：比较“节点引用”是否相同
                # 关键：一个指针回到头，另一个留在相遇点，同速前进
                p1, p2 = head, slow
                while p1 is not p2:
                    p1 = p1.next
                    p2 = p2.next
                return p1  # 环入口

        return None  # 无环
```

链表：`3 -> 2 -> 0 -> -4`，并且 `-4.next` 指向节点 `2`（pos = 1），环入口是值为 `2` 的节点。

- 判环阶段：`slow` 和 `fast` 会在环内某个节点相遇（例如在 `-4`）
- 找入口阶段：令 `p1` 回到 `3`，`p2` 留在相遇点，然后两者每次走一步：
  - 走若干步后会在节点 `2` 相遇
    所以返回入口节点 `2`。

## [21. 合并两个有序链表](https://leetcode.cn/problems/merge-two-sorted-lists/)

核心思路（双指针 + 哑节点 dummy）

两条有序链表 `l1、l2`，用指针逐个比较当前节点值，把较小的接到结果链表尾部。

关键技巧：用一个 **dummy 哑节点** 作为结果链表头的前驱，避免处理“第一个节点是谁”的分支。

时间 `O(m+n)`，空间 `O(1)`（不算新链表节点；这里只是重连指针）。

```python
class Solution:
    def mergeTwoLists(self, list1: 'ListNode', list2: 'ListNode') -> 'ListNode':
        dummy = ListNode(-1)
        tail = dummy

        while list1 and list2:
            # 关键：每次把较小节点接到 tail 后，并推进对应链表指针
            if list1.val <= list2.val:
                tail.next = list1
                list1 = list1.next
            else:
                tail.next = list2
                list2 = list2.next

            tail = tail.next  # 易错点：别忘了推进 tail

        # 关键：剩余部分直接接上（其中一个已空）
        tail.next = list1 if list1 else list2

        return dummy.next
```

## [2. 两数相加](https://leetcode.cn/problems/add-two-numbers/)

核心思路（逐位相加 + **进位 carry**）

题目给两个链表，按**逆序**存储数字（个位在头）。要求返回它们的和，结果也用同样方式返回。

做法：

- 用指针 `p1、p2` 分别遍历两链表
- 每一位相加：`x + y + carry`
- 当前位值：`sum % 10`，新进位：`sum // 10`
- 用 `dummy` 哑节点方便拼接结果
- 循环直到 `p1、p2` 都结束且 `carry == 0`

时间 `O(max(m,n))`，空间 `O(1)`（不算输出链表）。

```python
class Solution:
    def addTwoNumbers(self, l1: 'ListNode', l2: 'ListNode') -> 'ListNode':
        dummy = ListNode(0)
        tail = dummy
        carry = 0

        while l1 or l2 or carry:# 循环直到 `p1、p2` 都结束且 `carry == 0`
            x = l1.val if l1 else 0
            y = l2.val if l2 else 0

            s = x + y + carry
            carry = s // 10            # 关键：更新进位
            tail.next = ListNode(s % 10)
            tail = tail.next           # 易错点：别忘了推进 tail

            l1 = l1.next if l1 else None
            l2 = l2.next if l2 else None

        return dummy.next
```

`l1 = 2 -> 4 -> 3` 表示 342
`l2 = 5 -> 6 -> 4` 表示 465

逐位相加：

- 2 + 5 = 7，carry=0
- 4 + 6 = 10 → 写 0，carry=1
- 3 + 4 + 1 = 8，carry=0

结果：`7 -> 0 -> 8`（表示 807）

## [19. 删除链表的倒数第 N 个结点](https://leetcode.cn/problems/remove-nth-node-from-end-of-list/)

核心思路（**快慢指针 + dummy**）

用一个 `dummy` 指向 `head`，这样即使删除的是头节点也好处理。

步骤：

1. `fast` 先走 `n` 步
2. 然后 `fast` 和 `slow` 一起走，直到 `fast` 到达末尾（`None`）
3. 此时 `slow` 正好停在待删除节点的前一个节点
   删除：`slow.next = slow.next.next`

时间 `O(L)`，空间 `O(1)`。

```python
class Solution:
    def removeNthFromEnd(self, head: 'ListNode', n: int) -> 'ListNode':
        dummy = ListNode(0, head)  # 关键：用 dummy 统一处理删除头结点
        slow = fast = dummy

        # 关键：fast 先走 n 步，形成间距
        for _ in range(n):
            fast = fast.next

        # 易错点：这里条件是 fast.next，不是 fast
        while fast.next:
            fast = fast.next
            slow = slow.next

        # 删除 slow 的下一个节点（倒数第 n 个）
        slow.next = slow.next.next

        return dummy.next
```

## [543. 二叉树的直径](https://leetcode.cn/problems/diameter-of-binary-tree/)

核心思路（深度优先搜索 DFS）

二叉树的直径是树中任意两节点之间的最长路径长度。该路径不一定经过根节点。

关键点：

- 直径是由一条路径上的节点决定的，路径可能是从一个叶子节点到另一个叶子节点。
- 每个节点的直径由它的左子树和右子树的深度决定：`diameter = left_depth + right_depth`。
- 对每个节点递归计算其左子树和右子树的深度，并更新最大直径。

递归过程：

- 深度计算：`depth(node) = max(depth(node.left), depth(node.right)) + 1`
- 直径计算：`diameter(node) = left_depth + right_depth`

我们需要同时**返回每个节点的最大深度**，**并更新最大直径**。

时间复杂度：`O(n)`（每个节点只访问一次）
空间复杂度：`O(h)`（递归栈深度，最坏情况为 `O(n)`）

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def diameterOfBinaryTree(self, root: 'TreeNode') -> int:
        self.max_diameter = 0 # 全局变量记录
        
        def depth(node: 'TreeNode') -> int:
            if not node:
                return 0
            
            # 递归计算左右子树的深度
            left_depth = depth(node.left)
            right_depth = depth(node.right)
            
            # 更新最大直径
            self.max_diameter = max(self.max_diameter, left_depth + right_depth)
            
            # 返回当前节点的深度
            return max(left_depth, right_depth) + 1
        
        depth(root)
        return self.max_diameter # 有返回值
```

输入：

```
     1
    / \
   2   3
  / \
 4   5
```

递归过程：

- 对节点 4、5，`depth(4) = 1, depth(5) = 1`
- 对节点 2，`left_depth = 1, right_depth = 1`，更新最大直径为 `2`
- 对节点 3，`depth(3) = 1`（右子树）
- 对根节点 1，`left_depth = 2, right_depth = 1`，更新最大直径为 `3`

最终返回 `3`，即 `4 -> 2 -> 1 -> 3` 或 `5 -> 2 -> 1 -> 3` 是最长路径。

输出：3

```python
class Solution:
    def levelOrder(self, root: 'TreeNode') -> 'List[List[int]]':
        if not root:
            return []

        result = []
        queue = deque([root])

        while queue:
            level = []
            level_size = len(queue)

            # 遍历当前层

                node = queue.popleft()  # 弹出队列中的节点

                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)

            result.append(level)

        return result
```

## [108. 将有序数组转换为二叉搜索树](https://leetcode.cn/problems/convert-sorted-array-to-binary-search-tree/)

核心思路（**递归，分治法**）

将有序数组转换为二叉搜索树（BST），我们可以利用分治法，选取数组的**中间元素作为根节点**，然后**递归地构建左子树和右子树**。

递归思路：

- 中间元素是当前子数组的根节点。
- 中间元素左侧部分构成左子树，右侧部分构成右子树。
- 递归处理每一部分，直到数组为空。

选择中间元素的好处是，保证每次选择的根节点左右两边的元素个数相近，从而保证生成的树是平衡的（尽量避免退化成链表）。

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def sortedArrayToBST(self, nums: List[int]) -> 'TreeNode':
        if not nums:
            return None
        
        mid = len(nums) // 2  # 关键：选择中间元素作为根节点
        
        # 创建当前根节点
        root = TreeNode(nums[mid])
        
        # 递归构建左子树和右子树
        root.left = self.sortedArrayToBST(nums[:mid])   # 左半部分构建左子树
        root.right = self.sortedArrayToBST(nums[mid+1:])  # 右半部分构建右子树
        
        return root
```

```python
    def sortedArrayToBST(self, nums: List[int]) -> Optional[TreeNode]:
        def build(l, r, nums:List[int]):
            if l > r:
                return None
            mid = (l + r) // 2
            root = TreeNode(nums[mid])
            root.left = build(l, mid - 1, nums)
            root.right = build(mid + 1, r, nums)
            return root
        return build(0, len(nums) - 1, nums)
```

## [437. 路径总和 III](https://leetcode.cn/problems/path-sum-iii/)

有题解

核心思路（前缀和 + 深度优先搜索 DFS）

题目要求找出二叉树中路径和为目标值的路径。路径可以从任意节点开始，且路径不一定是从根节点开始的。

关键点：

- **路径和**：路径是从节点到节点的路径，**不一定是从根节点开始**，而是可以在树的任意位置开始。
- 这道题涉及到二叉树中的**所有路径**，不仅仅是从根节点开始的路径。

方法：

1. **前缀和（递归 + 深度优先搜索）**：
   - 在递归过程中，我们计算从当前节点到所有其他节点的路径和，查看是否存在符合目标和的路径。
   - 可以通过一个哈希表记录每个路径和出现的次数，方便快速查找。
   - 对于每个节点，查看从这个节点出发的路径和是否能满足条件。
2. **路径总和**：
   - 在遍历到每个节点时，查看从当前节点开始的路径总和。
   - 为了避免多次计算，可以使用一个辅助函数，递归地检查从当前节点出发的路径和是否等于目标值。

```python
class Solution:
    def pathSum(self, root: 'TreeNode', target: int) -> int:
        # 哈希表，记录从根到当前节点的路径和
        prefix_sum = {0: 1}  # 初始化，路径和为0的路径数量为1
        self.result = 0

        def dfs(node, current_sum):
            if not node:
                return
            
            # 更新当前路径和
            current_sum += node.val

            # 如果当前路径和减去目标值在哈希表中存在，说明有符合条件的路径
            self.result += prefix_sum.get(current_sum - target, 0)

            # 将当前路径和存入哈希表
            prefix_sum[current_sum] = prefix_sum.get(current_sum, 0) + 1

            # 递归遍历左右子树
            dfs(node.left, current_sum)
            dfs(node.right, current_sum)

            # 回溯：递归结束后，要减少当前路径和的计数
            prefix_sum[current_sum] -= 1

        # 从根节点开始DFS
        dfs(root, 0)
        return self.result
```

输入：

```
      10
     /  \
    5   -3
   / \    \
  3   2   11
 / \   \
3  -2   1
```

目标和：`8`

过程：

1. 从节点 `10` 开始，递归遍历每个路径的和，检查是否存在符合目标的路径。
2. 通过前缀和的记录，我们发现从节点 `5` 到 `3`，路径和为 `8`。
3. 继续递归遍历其他路径，找出所有符合条件的路径。

最终，找到路径 `[5, 3]` 和 `[10, -3, 11]`，共计 `2` 条路径和为 `8`。

输出：`2`

## [124. 二叉树中的最大路径和](https://leetcode.cn/problems/binary-tree-maximum-path-sum/)

有题解

递归过程：

1. **路径的定义**：
   - 以任意节点为起点的路径，可以是从该节点向下走的路径（可以经过其子节点，但不能回到父节点）。
   - 该路径和只包括节点本身的值和其下方节点的路径和。
2. **状态定义**：
   - 对每个节点，我们有两种选择：
     1. 以该节点为根，计算其贡献给路径的最大和。
     2. 以该节点为根，并尝试加入其左右子树的路径（在一侧更大时，选择该侧）。
3. **递归步骤**：
   - 递归地计算每个子树的最大路径和，并根据情况更新全局最大值（包括子树返回的最大值和可能通过该节点的路径）。
4. **计算每个节点的最大路径和**：
   - 以该节点为根，节点可以是路径的起点、终点或中间点。对于该节点的路径：
     - 要么仅包含当前节点和左子树路径，要么仅包含当前节点和右子树路径，要么包含当前节点及其左、右子树路径。
5. **返回节点的最大贡献**：
   - 对于每个节点，我们返回的是：`max(node.val, node.val + max(left, right))`，即当前节点加上其左右子树的最大贡献。

```python
class Solution:
    def maxPathSum(self, root: 'TreeNode') -> int:
        self.max_sum = float('-inf')  # 初始化最大路径和为负无穷
        
        # 递归计算路径和
        def helper(node):
            if not node:
                return 0
            
            # 递归计算左子树的最大路径和
            left = max(helper(node.left), 0)  # 负值不计入路径
            right = max(helper(node.right), 0)  # 负值不计入路径
            
            # 更新全局最大路径和：包含当前节点和其左右子树的路径
            self.max_sum = max(self.max_sum, left + right + node.val)
            
            # 返回当前节点的最大贡献：要么是左子树和当前节点，要么是右子树和当前节点
            return node.val + max(left, right)
        
        # 开始递归计算
        helper(root)
        return self.max_sum
```

输入：

```
      1
     / \
    2   3
   / \
  4   5
```

过程：

1. 从节点 `4` 和 `5` 开始：
   - `4` 贡献 `4`
   - `5` 贡献 `5`
2. 节点 `2` 贡献：`2 + 4 + 5 = 11`
3. 节点 `3` 贡献：`3`
4. 最终结果：`4 + 2 + 5 = 11` 是路径和的最大值。

输出：`11`

## [169. 多数元素](https://leetcode.cn/problems/majority-element/)

**摩尔投票法**是一个**线性时间**、**常数空间**的算法，利用了多数元素的性质：**如果一个元素的出现次数大于数组长度的一半，那么它一定是最后剩下的那个候选元素**。

**算法步骤**：

1. **初始化候选元素**和计数器 `count`。
2. **遍历数组**：
   - 如果 `count == 0`，更新候选元素为当前元素，`count = 1`。
   - 如果当前元素等于候选元素，`count++`。
   - 如果当前元素不等于候选元素，`count--`。
3. **返回候选元素**。由于题目保证存在多数元素，最终候选元素就是所求结果。

```python
class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        candidate, count = None, 0
        
        for num in nums:
            if count == 0:
                candidate, count = num, 1
            elif num == candidate:
                count += 1
            else:
                count -= 1
        
        return candidate
```

输入：

```
nums = [3, 2, 3]
```

过程：

1. 初始 `candidate = None`, `count = 0`
2. 第一个元素 `3`，`count = 0`，更新候选元素为 `3`，`count = 1`
3. 第二个元素 `2`，`count = 1`，`3 != 2`，`count = 0`
4. 第三个元素 `3`，`count = 0`，更新候选元素为 `3`，`count = 1`
5. 遍历结束，候选元素是 `3`。

输出：`3`

------

说明：

- **摩尔投票法**通过一个计数器动态调整候选元素，利用了多数元素会在遍历过程中“压倒”其他元素的特性，极大地优化了时间和空间复杂度。

## [75. 颜色分类](https://leetcode.cn/problems/sort-colors/)

核心思路（三指针法）

题目要求将包含 `0, 1, 2` 的数组排序，使得 `0`、`1`、`2` 分别排在前、中、后三部分。

我们可以用 **三指针法**（也称为 **荷兰国旗问题**）来高效地完成这个任务。

三指针法：

- 我们用三个指针：
  - `low`：指向当前需要放 `0` 的位置
  - `mid`：当前遍历的元素
  - `high`：指向当前需要放 `2` 的位置
- 遍历数组并进行如下操作：
  - `nums[mid] == 0`：交换 `nums[mid]` 和 `nums[low]`，然后 `low` 和 `mid` 都向右移动一位。
  - `nums[mid] == 1`：`mid` 向右移动一位。
  - `nums[mid] == 2`：交换 `nums[mid]` 和 `nums[high]`，然后 `high` 向左移动一位，但 `mid` 不动（因为我们还需要检查交换过来的元素）。

通过这种方式，数组中的 `0` 会被放到前面，`2` 放到后面，`1` 自然排在中间。

```python
class Solution:
    def sortColors(self, nums: List[int]) -> None:
        low, mid, high = 0, 0, len(nums) - 1
        
        while mid <= high:
            if nums[mid] == 0:
                nums[low], nums[mid] = nums[mid], nums[low]
                low += 1
                mid += 1
            elif nums[mid] == 1:
                mid += 1
            else:  # nums[mid] == 2
                nums[mid], nums[high] = nums[high], nums[mid]
                high -= 1
        
```

输入：

```
nums = [2, 0, 2, 1, 1, 0]
```

过程：

1. 初始状态：`low = 0, mid = 0, high = 5`
2. `nums[mid] == 2`，交换 `nums[0]` 和 `nums[5]`，`high -= 1`，状态：`nums = [0, 0, 2, 1, 1, 2]`，`high = 4`
3. `nums[mid] == 0`，交换 `nums[0]` 和 `nums[0]`，`low += 1`，`mid += 1`，状态：`nums = [0, 0, 2, 1, 1, 2]`，`low = 1, mid = 1`
4. `nums[mid] == 0`，交换 `nums[1]` 和 `nums[1]`，`low += 1`，`mid += 1`，状态：`nums = [0, 0, 2, 1, 1, 2]`，`low = 2, mid = 2`
5. `nums[mid] == 2`，交换 `nums[2]` 和 `nums[4]`，`high -= 1`，状态：`nums = [0, 0, 1, 1, 2, 2]`，`high = 3`
6. `nums[mid] == 1`，`mid += 1`，状态：`nums = [0, 0, 1, 1, 2, 2]`，`mid = 3`
7. `nums[mid] == 1`，`mid += 1`，结束，状态：`nums = [0, 0, 1, 1, 2, 2]`

输出：`[0, 0, 1, 1, 2, 2]`

------

总结：

- **三指针法**是一种高效的排序方法，特别适用于只包含三种不同元素的情况。
- 它通过两个指针 (`low` 和 `high`) 分别指向需要放 `0` 和 `2` 的位置，`mid` 指向当前遍历的元素。通过交换并适时调整指针的位置，可以一次性将数组分成三部分。

## [287. 寻找重复数](https://leetcode.cn/problems/find-the-duplicate-number/)

核心思路（快慢指针 / Floyd 判圈）

题目要求在一个包含 `n + 1` 个整数的数组中找到一个重复的数字，数组中的元素是从 1 到 `n` 之间的整数。要求 **不修改数组**，并且在 **O(n)** 时间和 **O(1)** **空间**的情况下找到重复的数字。

方法：**快慢指针法（Floyd 判圈）**

这个问题可以通过 **快慢指针** 来解决。思路类似于 **环形链表** 的问题，具体步骤如下：

1. **数组模拟链表**：将数组的元素看作是一个链表的指向关系，即元素的值为下标指向的节点。
2. **快慢指针**：
   - 快指针每次移动 2 步，慢指针每次移动 1 步。
   - 如果数组中存在重复的数字，那么它们在链表中的路径必定会形成一个环。
   - 快慢指针最终会在环的某个位置相遇。
3. **找环的入口**：
   - 当快慢指针相遇时，保持一个指针在相遇点，另一个指针从头开始，每次都向前移动 1 步。
   - 最终它们会在环的入口处相遇，这个入口就是重复数字的位置。

```python
from typing import List
class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        s = f = 0
        while True:
            s = nums[s]
            f = nums[nums[f]]
            if s == f:
                break
        f = 0
        while s != f:
            s = nums[s]
            f = nums[f]
        return s
```

## [560. 和为 K 的子数组](https://leetcode.cn/problems/subarray-sum-equals-k/)

核心思路（前缀和 + 哈希表）

题目要求在一个整数数组中找到和为 `k` 的子数组数量。我们可以使用 **前缀和** 和 **哈希表** 的结合来高效解决这个问题。

关键点：

- **前缀和**：从数组的第一个元素到当前元素的和。
- 如果我们已经知道了从 `i` 到 `j` 的子数组和是 `sum(j) - sum(i) = k`，那么我们只需要查找哈希表中 `sum(i)` 是否已经存在，若存在，则说明找到了一个和为 `k` 的子数组。

方法：

1. 使用一个哈希表 `prefix_sum_count`，记录每个前缀和出现的次数。
2. 初始时，哈希表中有一个 `{0: 1}`，表示前缀和为 `0` 的次数是 `1`。
3. 遍历数组，计算当前前缀和 `current_sum`：
   - 计算 `current_sum - k`，查找是否有这样的前缀和出现过，如果有，说明存在一个子数组的和为 `k`，然后计数。
   - 将当前的 `current_sum` 存入哈希表。

```python
from collections import defaultdict

class Solution:
    def subarraySum(self, nums: List[int], k: int) -> int:
        prefix_sum_count = defaultdict(int)
        prefix_sum_count[0] = 1  # 初始时，前缀和为0出现一次
        current_sum = 0
        count = 0
        
        for num in nums:
            current_sum += num  # 更新当前的前缀和
            
            # 如果 (current_sum - k) 已经在哈希表中，说明有子数组和为 k
            if current_sum - k in prefix_sum_count:
                count += prefix_sum_count[current_sum - k]
            
            # 将当前的前缀和添加到哈希表(必须放在这，不能放在前面)
            prefix_sum_count[current_sum] += 1
        
        return count
```

输入：

```
nums = [1,1,1], k = 2
```

过程：

1. 初始时：`prefix_sum_count = {0: 1}`, `current_sum = 0`, `count = 0`
2. 遍历 `nums`：
   - 第一个 `1`：`current_sum = 1`，`current_sum - k = 1 - 2 = -1`，`prefix_sum_count[-1] = 0`，不符合条件。更新 `prefix_sum_count = {0: 1, 1: 1}`。
   - 第二个 `1`：`current_sum = 2`，`current_sum - k = 2 - 2 = 0`，`prefix_sum_count[0] = 1`，符合条件，`count += 1`。更新 `prefix_sum_count = {0: 1, 1: 1, 2: 1}`。
   - 第三个 `1`：`current_sum = 3`，`current_sum - k = 3 - 2 = 1`，`prefix_sum_count[1] = 1`，符合条件，`count += 1`。更新 `prefix_sum_count = {0: 1, 1: 2, 2: 1, 3: 1}`。
3. 最终 `count = 2`，所以结果是 `2`。

输出：`2`

- **前缀和 + 哈希表**的结合提供了一个非常高效的方法来解决这个问题。通过维护当前的前缀和和记录每个前缀和出现的次数，我们可以在 `O(n)` 时间内计算出和为 `k` 的子数组数量。

## [239. 滑动窗口最大值](https://leetcode.cn/problems/sliding-window-maximum/)

核心思路（双端队列）

关键点：

1. 每次滑动窗口，移除窗口左侧的元素，加入窗口右侧的新元素，计算当前窗口的最大值。
2. 为了高效计算窗口的最大值，可以使用 **双端队列（Deque）**：
   - 保持队列中的元素是递减的。队列头部的元素始终是当前窗口的最大值。
   - 每次滑动窗口时，移除已经不再在窗口内的元素（即窗口左边的元素），同时将新元素加入队列并保持递减顺序。

方法：

1. 使用 **双端队列（Deque）** 来存储数组的索引。队列的元素从队首到队尾是递减的，这样队首的元素总是当前窗口的最大值。
2. 遍历数组，逐步更新队列：
   - 移除不在窗口内的元素。
   - 移除队列中比当前元素小的元素，以保持队列中元素的递减顺序。
3. 在每次窗口滑动时，队列头部的元素就是当前窗口的最大值。

先移除左， 再移除右， 加进去， 加结果

```python
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        # 结果数组
        result = []
        # 双端队列：存储当前窗口的索引，队列中元素递减
        dq = deque()
        
        for i in range(len(nums)):
            # 移除不在当前窗口内的元素 注意范围
            if dq and dq[0] < i - k + 1:
                dq.popleft()
            
            # 移除队列中小于当前元素的索引（保持递减顺序）
            while dq and nums[dq[-1]] < nums[i]:
                dq.pop()
            
            # 注意： 将当前元素的索引加入队列
            dq.append(i)
            
            # 当遍历到至少 `k` 个元素时，添加窗口最大值到结果数组
            if i >= k - 1:
                result.append(nums[dq[0]])  # 队头元素是当前窗口的最大值
        
        return result
```

输入：

```
nums = [1,3,-1,-3,5,3,6,7], k = 3
```

过程：

1. **初始化**：`dq = []`，`result = []`。
2. 逐步滑动窗口：
   - 1st window `[1, 3, -1]`：最大值是 `3`，添加到结果，`result = [3]`。
   - 2nd window `[3, -1, -3]`：最大值是 `3`，添加到结果，`result = [3, 3]`。
   - 3rd window `[-1, -3, 5]`：最大值是 `5`，添加到结果，`result = [3, 3, 5]`。
   - 4th window `[-3, 5, 3]`：最大值是 `5`，添加到结果，`result = [3, 3, 5, 5]`。
   - 5th window `[5, 3, 6]`：最大值是 `6`，添加到结果，`result = [3, 3, 5, 5, 6]`。
   - 6th window `[3, 6, 7]`：最大值是 `7`，添加到结果，`result = [3, 3, 5, 5, 6, 7]`。

输出：`[3, 3, 5, 5, 6, 7]`

```python
class Solution:
    def minWindow(self, s: str, t: str) -> str:
        if not s or not t:
            return ''
        minl = float('+inf')
        need = Counter(t)
        window = defaultdict(int)
        valid = 0
        l = r = 0
        start = 0
        while r < len(s):
            c = s[r]
            r += 1
            if c in need:
                window[c] += 1
                if window[c] == need[c]:
                    valid += 1

            while len(need) == valid:
                if r - l < minl:
                    start = l
                    minl = r - l

                d = s[l]
                l += 1
                if d in need:
                    if need[d] == window[d]:
                        valid -= 1
                    window[d] -= 1
        if minl == float('+inf'):
            return ""
        return s[start: start + minl]
```

```python
class Solution:
    def numIslands(self, grid: List[List[str]]) -> int:
        self.m = len(grid)
        self.n = len(grid[0])
        self.count = 0
        direct = [[0, 1], [1, 0], [-1, 0], [0, -1]]

            dq.append([i, j])

                    self.count += 1
                    bfs(grid, i, j)
```

```python
            partition2 = (m + n + 1) // 2 - partition1
            
        # 如果 partition1 为 0，则左边的元素是负无穷；如果 partition1 为 m，则右边的元素是正无穷
            maxLeft1 = float('-inf') if partition1 == 0 else nums1[partition1 - 1]
            minRight1 = float('inf') if partition1 == m else nums1[partition1]
            
            maxLeft2 = float('-inf') if partition2 == 0 else nums2[partition2 - 1]
            minRight2 = float('inf') if partition2 == n else nums2[partition2]
            
            # 满足条件时，计算中位数
            if maxLeft1 <= minRight2 and maxLeft2 <= minRight1:
                if (m + n) % 2 == 0:
                    return (max(maxLeft1, maxLeft2) + min(minRight1, minRight2)) / 2
                else:
                    return max(maxLeft1, maxLeft2)
            elif maxLeft1 > minRight2:
                right = partition1 - 1
            else:
                left = partition1 + 1
        
        raise 0.0 # 不会执行到这
```
